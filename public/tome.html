On oOoo*

  1. There is no oOoo*

On gateways

  1. Gateways are encrypted TrueCrypt volumes shared on the local network.
  2. Source gateways are used to proffer (share) artifacts (files) to other cultists (users)
    • Used in a read only mode.
    • Opened regularly to scan for newly proffered artifacts.
    • Artifacts are available for other cultists to clone only while the original is available in your source gateway.
    • Some artifacts are ignored:
      • Any files under the following special folders: 
        /clones/
        or 
        /ignored/
      • Any hidden files (prefixed with a "." character), for example 
        /.Trash-41231/Deleted-file.txt
        or 
        /.settings
      • Known system files, for example 
        /System Volume Information/
        or 
        /Recycled
  3. Sink gateways are used to clone (copy) artifacts (files) that were proffered (shared) by other cultists (users).
    • Used in a write only mode.
    • Opened only to clone artifacts.
    • Cloned artifacts are placed under the folder 
      /clones/
  4. In this way artifacts are never stored on disk in plain-text.
  5. Cultists should never locally mount a gateway TrueCrypt volume while oOoo* has the gateway open. To do so may cause corruption.

On creating a gateway

  1. Create a (windows) network share, with appropriate permissions...
    • Sink: The share must allow write access to "Everyone" if you wish to receive clones.
    • Source: The share must allow read access to "Everyone".
  2. Create a TrueCrypt 7 encrypted volume in that share, using the "create volume" wizard following the instructions below:
    1. Create an encrypted file container
    2. Standard TrueCrypt volume
    3. Volume location should be in the network-share created earlier, with any name. The name should be either obscure or the name of commonly large files such as: 
      pc678.vmdk
      or 
      install.GHO
      or 
      hiberfil.sys
    4. Default encryption options
    5. Volume size should be fixed (as variable sized volumes can cause corruption), and recommend >10GB size if possible (it will fill fast)
    6. I will not store files larger than 4GB on the volume. (Only on Ubuntu as this enables selection of FAT in the next step).
    7. FAT file system
    8. Format
  3. Ensure the encrypted volume file has appropriate permissions...
    • Sink: read-write permissions
    • Source: read-only permissions
  4. In oOoo* add a gateway with these details, selecting the correct mode, source or sink, depending on the use.

On avoiding insanity

  1. Cultists using Ubuntu should ensure that when removing unwanted files from a gateway that they permanently remove (e.g. shift-delete) the files, as otherwise the file is just moved to a hidden .Trash directory in the gateway root directory.
  2. Cultists using Windows 7 may have clones fail repeatedly. If so this is likely due to a known Windows 7 issue. Please refer to http://alan.lamielle.net/2009/09/03/windows-7-nonpaged-pool-srv-error-2017
  3. Gateways on USB disks may occassionaly fail due to timeout in data send/receive. Recommend using USB 3.0 devices in a USB 3.0 SS compatible port (look for SS mark on USB port, super speed).
  4. Cultists code-names are plucked at random from the ether.
